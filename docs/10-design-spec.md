# Дизайн-спека редизайна Logic Thread (реализована в src/client/index.html + main.ts)

Синтез 4 исследований (daily-games, dark-a11y, Devvit, type/spacing) и 4 аудитов (color, type-size, hierarchy, mobile). Все контрасты пересчитаны по WCAG relative luminance на реальных подложках.

Главные принципы (не меняются): грид доминирует; dark navy + мягкий янтарный акцент #ff8c42 (НЕ переходить на #FF4500 — занят системным UI Reddit); green = подтверждено, red = невозможно/нарушено; мобильный приоритет (tap-only, iOS webview); один self-contained HTML; JS-логику не трогаем, DOM — минимально (классы/обёртки).

> **Обновление (2026-07-07):** community-loop (голос за тему / submit подозреваемого / deduction-card для шеринга) и альтернативный вид поля «Вкладки» удалены из прототипа; остался единственный вид — single board. Относящиеся к ним директивы ниже (poll/submit/viewswitch/tab/cell/red-soft/tomorrow-блок) — **исторические**, читать как аудит-контекст, не как задачи.

### 1.2 Type scale — 7 ступеней, целые px, всё через var()

| Токен | px | LH | Роль (замены) |
|---|---|---|---|
| `--fs-2xs` | 11 | 1.2 | ТОЛЬКО uppercase-микролейблы с letter-spacing .5–1px, weight 600+: заголовки панелей, hint-tag, badge, .by, .cnt (10/10.5 → 11) |
| `--fs-xs` | 12 | 1.5 при переносе | подписи, счётчики, grid-help, footer, sub (11.5/12 → 12) |
| `--fs-sm` | 13 | 1.45–1.5 | базовый: УЛИКИ, ячейки, таблица результата (12.5/13 → 13) |
| `--fs-base` | 14 | 1.2 (flex-центрирование) | кнопки, заголовок дела, названия панелей (13.5/14/15 → 14) |
| `--fs-md` | 17 | 1.2 | иконки колонок, крупные числа, марки ячеек (17/21 → 17… марку ✕/✓ можно 20) |
| `--fs-lg` | 20 | 1.2 | логотип, h3-хиро статов результата (20/21 → 20) |
| `--fs-xl` | 24 | 1.2 | h2 экрана результата |

База: `body { font-size: 13px; line-height: 1.45; }`. Спец-исключение вне шкалы: `@media (pointer: coarse) { .submit-row input { font-size: 16px } }` — иначе iOS автозумит страницу при фокусе. `font-variant-numeric: tabular-nums` — на таймер, счётчики, poll-проценты, статы результата.

### 1.3 Spacing — 4pt-сетка

`--sp-1: 4px; --sp-2: 8px; --sp-3: 12px; --sp-4: 16px; --sp-5: 24px; --sp-6: 32px`. 2px допустим как микро-зазор. Маппинг текущих: 3,5→4; 6,7→8; 9,10,11→8/12; 13,14→12/16; 18→16; 22,26→24; border-spacing 5→4. Паддинг панелей: 16px desktop / 12px mobile.

### 1.4 Radius — 4 токена + правило вложенности

`--r-s: 8px` (chip-m, tab, viewswitch, HUD-чипы; 9→8), `--r-m: 12px` (cell, bcell, btn, toast, poll-opt, input, card; 10→12), `--r-l: 16px` (борд-фрейм, модалка результата; 18→16), `--r-pill: 999px` (diff, s-chip, номер улики вместо 50%). Вложенность: inner = outer − padding (карточки внутри модалки при внешнем 16px и padding 24px → углы --r-s или 0, не 12px).

### 1.5 Touch/misc

Минимумы: кнопки min-height 44px; строки улик min-height 44px на touch; тап-цели <44px расширять `@media (pointer:coarse) { .x { position:relative } .x::after { content:''; position:absolute; inset:-8px } }`. Всем интерактивам `touch-action: manipulation`. Глобально `:root { color-scheme: dark }` (тёмные UA-скроллбары/чекбоксы), `:focus-visible { outline: 2px solid var(--blue); outline-offset: 2px }`, `body { overflow-x: hidden }` (страховка).

## 2. Colorblind-флаиры: как выглядит чип

Голые эмодзи 🔴🔵🟢🟣 заменить на CSS-чип (`chipLabel` для flair, index.html:732; сетка :770-784):

- Круг (30×30 базово / не меньше 30 на мобиле), фон — цвет флаира из Okabe-Ito (`--flair-r/b/g/p`).
- Внутри — **буква R / B / G / P**, weight 700, `--fs-sm`, цвет `--on-accent #0b101e` (тёмная буква на всех четырёх — все фоны светлые, контраст ≥4.5:1). Буква — главный канал: имена флаиров и так Red/Blue/Green/Purple, работает при любом типе CVD и в grayscale.
- В тексте улик эмодзи можно оставить (там рядом слово «красного флаира» — дублирование уже есть), либо заменить на такой же inline-чип 14px для консистентности.
- В таблице «Итог расследования» — чип + слово, `white-space: nowrap`.

**Вычеркнутое состояние чипа (критично, чинит P0):** opacity `.55` (не .3), **убрать `grayscale`** полностью — цвет+буква должны остаться опознаваемыми; линию зачёркивания вынести из-под opacity: рисовать псевдоэлементом на родителе `.bcell` либо задать прямой цвет `--strike #6b7590`, 2px, без наследования прозрачности (3.05:1 против фона чипа). Итог: видно, ЧТО вычеркнуто и КАКОЙ это флаир.

## 3. Компонентные изменения

### P0 — блокеры (мобайл ломается или действие недоступно)

1. **Грид, мобильный скролл** → `.grid-wrap { justify-content: flex-start }` в MQ (сейчас center + overflow обрезает имена подозреваемых НЕДОСТИЖИМО, scrollLeft max 66 из 132); имена закрепить: `th.rowh { position: sticky; left: 0; background: var(--surface-1); z-index: 2 }`. На десктопе центрирование через `table { margin: 0 auto }` (авто-маргины схлопываются при переполнении). Скроллбар: `scrollbar-width: thin; scrollbar-color: var(--border) transparent` + webkit-аналог (height 6px) — убирает белую системную полосу. Аффорданс обрезки: правый edge-fade `mask-image: linear-gradient(90deg, #000 calc(100% - 18px), transparent)`, снимать классом при доскролле.
2. **Шапка/HUD мобайл** → `.top { flex-wrap: wrap; row-gap: var(--sp-1) }`, `.brand .sub { display: none }` в MQ, `.hud { width: 100%; justify-content: flex-end; gap: 6px }`, чипы `padding: 4px 8px; font-size: var(--fs-xs)` — сейчас HUD вылезает на 110px за край и прячет прогресс и подсказки.
3. **Подтверждение без правого клика (iOS tap-only)** → для `pointer: coarse` добавить long-press (pointerdown + ~450ms → applyMove(...,2), отмена по pointerup/pointermove) и/или tap-цикл `· → ✕ → ✓ → ·` в режиме «Вкладки»; текст grid-help подставлять по `matchMedia('(pointer:coarse)')`: «тап — вычеркнуть · тап по пульсирующему — ✓» (одна строка), убрав «правый клик». Это минимальная JS-правка обработчиков ввода, не логики.
4. **Вычеркнутые чипы «Одного борда»** → см. раздел 2: opacity .3+grayscale(.9) (контраст линии 1.43:1, чип vs фон 1.02:1) заменить на opacity .55 без grayscale + линия `--strike` вне opacity + буква в чипе.
5. **Флаир-чипы CVD** → см. раздел 2 (буквы R/B/G/P на Okabe-Ito фонах).

### P1 — иерархия, читаемость, тач

6. **Оранжевый = только CTA.** Единственная залитая оранжевым кнопка на игровом экране — «Проверить дело». Заголовок «УЛИКИ» → `--muted` uppercase `--fs-2xs`; `.tab.active` → фон `--surface-2` + оранжевый текст + 2px нижняя рамка `--orange` (вместо заливки, дублирующей CTA); accent-color чекбоксов ассистента → `--muted`; активный viewswitch → фон `--surface-2`, рамка `--border-active`, текст `--text` (сейчас он делит тёплую пару #2a2312+yellow с hint-семантикой). CASE #N остаётся оранжевым — брендовое эхо и ритуальный номер выпуска.
7. **Состояние CTA = индикатор прогресса** (паттерн Connections): пока борд не 12/12 — «Проверить дело» в приглушённом состоянии (заливка `--orange-dim`, текст `--on-accent`, cursor default допустим — это не disabled в WCAG-смысле, но вести себя как сейчас); при 12/12 — полная заливка `--orange` + лёгкий pulse. Только CSS-класс от уже существующего счётчика.
8. **Controls: иерархия кнопок.** Primary = «Проверить дело» (filled `--orange`, текст `--on-accent`, min-height 44px). «Подсказка» = secondary (`--surface-2` + 1px `--border-active`). «Заново», «Отмена» = ghost (прозрачный фон, текст `--muted`); красный у «Заново» — только на hover/confirm. Мобайл: `.controls { flex-wrap: wrap; position: sticky; bottom: 0; z-index: 5; background: var(--surface-sunken) }`, primary `flex: 1 1 100%; order: 1; padding: 12px` (полная ширина нижней строкой = большая тап-зона), три служебные — рядом выше, `white-space: nowrap`.
9. **Viewswitch + assist: демоция в дев-панель.** Обернуть оба блока в одну строку `.devbar` с префиксом «⚙ playtest», `opacity: .6`, `font-size: var(--fs-2xs)`, dashed-рамка `--border`; переместить ПОД грид (или свернуть за иконку ⚙ с раскрытием по клику). Чекбоксы обернуть в label с padding ≥6px (тап по тексту). Кликабельное состояние сохраняет ≥3:1 — глушим прозрачностью группы, не цветом текста. Сейчас они стоят над сеткой, громче заголовков категорий и на мобиле разрастаются до 62px плиток.

## 4. Чего НЕ делать (анти-цели)

- **Viewswitch «Один борд/Вкладки» и режим «Вкладки» — УДАЛЕНЫ** (решение пользователя 2026-07-07): board победил вкладки в плейтестах, остался единственный вид. Аудит-пункты выше про demотацию viewswitch/devbar и тумблеры ассистента — историчны; вычёркивать в дев-панель больше нечего. (Прежняя директива «не удалять как плейтест-инструмент» отменена.)
- **Не менять JS-логику игры** (генерация, проверка, подсчёт). Разрешённый JS-минимум: обработчики ввода для touch (long-press/tap-цикл, toggle подсветки улики), подстановка текста grid-help, scrollIntoView, блокировка скролла оверлея.
- **Не переходить на Reddit-оранжевый #FF4500/#D93900** — отстройка от системного upvote-UI это преимущество.
- **Не заливать оранжевым большие площади** — акцент точечный (CTA, прогресс, бренд-эхо).
- **Не использовать `maximum-scale`/`user-scalable=no`** для борьбы с iOS-зумом — только 16px на инпуте.
- **Не ужимать тап-цели ради «влезания без скролла»** — 30px чип + hit-area + честный горизонтальный скролл лучше 26px чипов.
- **Не выражать состояние только цветом** — каждое (✕/✓/буква флаира/line-through/«!») дублируется знаком или формой.
- **Не вводить пятую ступень серого текста и дробные font-size** — 3 текстовых токена, 7 ступеней шкалы, целые px.
- **Не делать красным рутинное вычёркивание** одной интенсивности с ошибкой — насыщенный `--red` эксклюзивен для нарушений.
- **Не убирать эмодзи из текста улик** — там они дублированы словом и работают.
- **Порядок внедрения:** сначала объявить токены в `:root`, затем механическая замена значений (журналируемое решение для docs/09-design-decisions.md), затем компонентные P0→P1→P2; после каждого блока — скриншот-сверка с baseline.

## Отклонения при внедрении (после верификации, 4 агента раунда-2)

- **CTA всегда залитая** `--orange` (не «контур до 12/12 → заливка»): вариант «заливка --orange-dim + тёмный текст» не проходит 4.5:1 для 14px-текста, а контурный вариант читался как secondary. Состояние 12/12 передаётся только пульсацией `readypulse`; за модалкой анимация отключается (`body.modal-open`).
- **Линия зачёркивания флаир-чипов** — scoped `#99a1b8` (вместо глобального `--strike #6b7590`): на 30%-миксах флаиров `--strike` давал 2.07–2.65:1; `#99a1b8` даёт ≥3.69:1 на всех четырёх.
- **Добавлен токен `--fs-2xl: 28px`** для герой-стата результата (28px хардкод легализован в шкалу).
- **Глиф ⏱ (U+23F1)** без VS16 рендерится tofu в Firefox/webview — в HUD заменён на `⏱️` (с VS16), из герой-стата иконка убрана.
- **Вычеркнутый глиф**: opacity .6 (не .55 из спеки) — чтобы вычеркнутые цифры времени держали ~4.45:1.
- **Тап-подсветка «улика → поле» сознательно НЕ реализована**, хотя спека её требовала: механика подсветки поля убрана плейтестом (журнал решений, реш. 10) как путающая. Зафиксирована как открытый вопрос в [09-design-decisions.md](09-design-decisions.md).
- **Unknown-метка `#454f75`** (1.93:1) оставлена намеренно тихой — задизайненное состояние «нет данных».

## Итерация 2 (2026-07-07, по решениям пользователя)

**Inline-режим (игра в ленте, Devvit tall 512px).** `@media (max-height: 720px)`: шапка+casebar+controls зафиксированы, `.main` скроллится внутри (тонкий тёмный скроллбар); улики заполняют свою панель в двухколоночной раскладке, в stacked — окно 26vh; devbar и footer скрыты; порог 720px закрывает зону 620–720, где CTA уходила за фолд.

**Светлая карточка результата** (референс — r/ColorPuzzleGame): локальные переопределения токенов в скоупе `.result` (`#fff` фон, текст `#1c2437/#3c4660/#5f6b84`, зелёный `#147a43` ≥4.5:1, синий focus `#2563eb`, светлые тосты при `modal-open`, hover-затемнение вместо осветления). Карточка влезает в 900px-вьюпорт без прокрутки: статы одной строкой + PB/мировой (мок), гистограмма, свёрнутый итог, deduction card (кламп 2 строки), poll 2×2, submit, ряд retention-CTA (напомнить/архив/сабреддит/ещё раз — моки), закрытие ✕/Esc/клик по фону, повторный «Проверить дело» снова открывает карточку.

**Гистограмма времени решения** (дизайн по dataviz-методике): одна серия, бары ≤22px с зазором 4px и скруглённым верхом; нейтраль `#6b7590`, бар игрока `#d9631e` (валидатор: контраст ≥3:1 на белом, CVD ΔE 67.6); подписи выборочно — «вы» и пик (при совпадении — обе); текст подписей — чернильными токенами, не цветом серии; тултипы на бинах. Данные — мок `TIME_BINS`, в проде — с сервера.

**Тач-дополнение:** long-press по чипу борда показывает название кандидата (замена desktop-tooltip). Конфетти при решении — одноразовое, отключается `prefers-reduced-motion`.

## Итерация 3 (2026-07-07, «только игра»)

**Экран = пост.** `.app` 700px, `.board` — высота `min(512px, 100vh−16)` на десктопе: прототип по умолчанию выглядит как Devvit-пост в ленте (референс — r/ColorPuzzleGame). Бренд-строка, футер-примечание и легенда статусов удалены с игрового экрана; HUD сжат до «✔ прогресс · ✖ ошибки (появляется после первой) · ⏱ время» в строке дела; логотип — 🧵 перед CASE #N. Инструкция управления + легенда — тост за кнопкой «?». Controls: ↺ ↩ ? — иконки-ghost, 💡 Подсказка — secondary (на мобиле — иконка), CTA — единственная залитая.

**Десктоп 700×512 — без скроллов:** панель улик `clamp(210px,31%,240px)`, строки улик 12px/1.35 (все 10 видны), чипы 24px (bcell 44px) — грид входит в оставшиеся ~458px без горизонтальной прокрутки.

**Телефон — без скроллов:** новый компакт-рендер борда `renderBoardGridNarrow()` (переключение по `matchMedia(max-width:760px)`, display-only): имя подозреваемого над строкой из трёх ячеек, общая ширина ≤390px; колонки подписаны один раз сверху. Все улики + сетка + controls помещаются в 390×844 целиком. В ленточном превью 390×512 середина скроллится внутри поста (26vh улик) — полный вид открывается разворачиванием поста.

**Трейд-офф:** чипы 24px ниже рекомендуемых 44px тап-целей — компенсировано расширенной невидимой тап-зоной (::before) и long-press-названием; вернуться после плейтеста с телефона.
