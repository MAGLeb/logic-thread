# Геймплей и механики

## Logic grid

Основная сущность — **suspect**. Все остальные категории сопоставляются с suspect. Игрок решает всё в одном борде (строки — suspects, колонки — категории: Flair, Time, Object…), без вкладок. Огромную классическую grid со всеми подтаблицами НЕ показываем — тяжело для Reddit UI.

## Состояния ячейки

```
type CellState = "unknown" | "no" | "yes"
```

Основной вид поля — «один борд»: строки — suspects, колонки — категории, внутри ячейки чипы-кандидаты (проверено на прототипе, победил вкладки).

## Взаимодействие: чистый toggle (без подтверждения)

Главный принцип — **ничего не происходит само**. Каждое изменение на поле делает игрок; никаких каскадов, авто-вычёркиваний и авто-подтверждений (плейтесты 2–3: любая автоматизация, даже локальная, размывает понимание, что происходит).

В «одном борде» у чипа только два состояния — обычный и вычеркнутый:

- tap по кандидату — вычеркнуть / вернуть (чистый toggle: нажал = активировал / дезактивировал);
- отдельного «подтвердить» нет. Когда в ячейке остаётся один невычеркнутый кандидат, он и есть ответ (виден сам собой на фоне приглушённых вычеркнутых) — но остаётся обычным toggle-чипом: tap по нему просто вычёркивает его назад;
- undo отменяет ровно один ход (одну ячейку).

Вычёркивание — единственное действие. «Проверить дело» и счётчик n/12 засчитывают единственного оставшегося (невычеркнутого) кандидата за ответ — **«я всё вычеркнул» = «я решил»** (плейтесты 1 и 4). Все версии всегда видны, ничего не схлопывается: пока разгадываешь, важно видеть и вычеркнутые варианты (плейтест 4).

## Живой статус улик

Улики в списке подсвечиваются по текущим отметкам игрока:

- 🟢 выполнена;
- 🔴 нарушена (отметки противоречат улике — проверь ходы);
- серая — пока не ясно.

Статус считается от отметок игрока, а не от скрытого решения — честная обратная связь без спойлеров. Наведение/выбор улики НЕ подсвечивает и не обводит ячейки поля (плейтест 3: это путало) — улики и поле визуально независимы.

## Check solution

- всё верно → solved screen;
- есть неверный ответ (оставшийся кандидат противоречит решению) → +1 mistake, «something is inconsistent» (точную ошибку не раскрывать);
- не заполнено → «Not enough deductions yet».

## Hints (3 уровня, для MVP достаточно первого)

1. Подсветить полезный clue и связанные сущности.
2. Подсветить relevant row/column.
3. Поставить одну forced deduction.

## Обсуждение решения в комментах

Отдельной in-game карточки-механики нет. После решения игрок объясняет ход мысли органически в комментах к daily-посту — там же спорит и сверяет цепочку с другими детективами:

> 🧠 Clue 2 + Clue 5 доказывают: у Mira не могло быть жёлтого flair.

Комментарии — это доска дедукций и споров о решении, живёт нативно в треде Reddit.

## Состояния игры

1. **Not started** — case title, сразу grid и первый clue (без отдельного Start-экрана).
2. **In progress** — timer, mistakes, hints, active clue, grid, controls.
3. **Wrong check** — «Something does not fit the case. Mistakes: 2».
4. **Solved** — переход на result/community screen.
5. **Post-solve** — результат (время / серия / перцентиль), обсуждение в комментах, обратный отсчёт до нового дела, лидерборд.

Во время решения НЕ показывать: leaderboard, длинные списки. Всё это — награда после решения.
